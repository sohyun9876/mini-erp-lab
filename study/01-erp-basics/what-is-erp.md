# ERP란 무엇인가

---

## 1. ERP 정의

**ERP = Enterprise Resource Planning**

> 기업의 모든 자원을 통합 관리하여, 실시간으로 정보를 공유함으로써  
> 빠르고 정확하며 투명한 업무처리를 가능하게 해주는 시스템.

| 단어 | 의미 |
|------|------|
| **Enterprise** | 기업 전체 — 부서, 사업부, 법인 등 조직 전반 |
| **Resource** | 자원 — 인력, 자금, 재고, 설비, 시간 등 기업이 가진 모든 것 |
| **Planning** | 계획/관리 — 자원을 효율적으로 배분하고 흐름을 관리 |

---

## 2. ERP가 없으면 생기는 문제

ERP 이전에는 각 부서가 **자기만의 시스템(Excel, 개별 프로그램)**으로 데이터를 따로 관리했다.

### 대표적인 문제들

| 문제 | 설명 |
|------|------|
| **데이터 중복/불일치** | 영업팀 재고 수량 vs 창고팀 재고 수량이 다름 |
| **정보 단절** | A 부서에서 일어난 일을 B 부서가 실시간으로 알 수 없음 |
| **수작업 오류** | 부서 간 데이터를 사람이 옮기다가 실수 발생 |
| **의사결정 지연** | 경영진이 전사 현황을 파악하려면 각 부서에 취합 요청해야 함 |

### 실제 예시

```
❌ ERP 없을 때

영업팀  →  고객에게 납기 2주 약속하고 수주 확정
   ↓
창고팀  →  "재고 없음, 생산에 3주 걸림" 뒤늦게 통보
   ↓
결과   →  납기 지연, 고객 클레임, 신뢰도 하락
```

```
✅ ERP 있을 때

영업팀  →  수주 입력 시 실시간 재고/생산 일정 확인 가능
   ↓
결과   →  납기 약속 전 정확한 일정 확인 후 수주
```

## 3. ERP가 해결하는 것

### 데이터 통합
모든 부서가 **하나의 DB**를 사용한다.  
영업이 수주를 입력하면 재고, 생산, 회계 데이터가 동시에 연동된다.

### 실시간 정보 공유
어떤 부서에서 데이터가 바뀌면 **전사에 즉시 반영**된다.  
경영진도 별도 취합 없이 현재 상태를 실시간으로 확인할 수 있다.

### 프로세스 표준화
업무 흐름을 시스템이 강제하기 때문에 **사람마다 다르게 처리하는 문제**가 줄어든다.

## 4. ERP의 역사

```
1960s  MRP (Material Requirements Planning)
       └─ 생산 자재 수급 계획에 집중
          "언제, 얼마나 자재를 사야 하나?"

1980s  MRP II (Manufacturing Resource Planning)
       └─ 생산 전반으로 확장 (설비, 인력, 일정까지)
          "공장 전체를 어떻게 굴릴까?"

1990s  ERP (Enterprise Resource Planning)
       └─ Gartner가 1990년 처음 명명
          생산을 넘어 회계·인사·영업·구매까지 전사 통합
          SAP R/3, Oracle ERP 등장으로 대중화

2000s  대기업 중심 ERP 도입 확산
       └─ 도입 비용이 수십억~수백억 → 중소기업은 엄두 못 냄

2010s~ 클라우드 ERP / SaaS ERP 등장
       └─ 중소기업도 월정액으로 ERP 사용 가능해짐
```

## 5. ERP 핵심 모듈 구조

### 모듈이란?
ERP는 업무 영역별로 기능을 나눈 **모듈(Module)** 단위로 구성된다.  
각 모듈은 독립적으로 동작하지만, 데이터는 서로 연결된다.

### 모듈을 나누는 이유

| 이유 | 설명 |
|------|------|
| **전문화** | 각 업무 영역의 복잡한 로직을 집중해서 처리 |
| **도입 유연성** | 필요한 모듈만 골라서 도입 가능 |
| **유지보수 용이** | 한 모듈 수정이 다른 모듈에 영향 최소화 |
| **유기적 연결** | 모듈 간 데이터는 자동 연동되어 이중 입력 불필요 |

### 핵심 모듈

| 모듈 | 담당 업무 |
|------|-----------|
| **회계 (FI)** | 전표 입력, 원장 관리, 재무제표, 세금계산서 |
| **인사/급여 (HR)** | 사원 관리, 근태, 급여 계산, 퇴직금 |
| **재고/물류 (IM)** | 입출고, 재고 실사, 창고 관리 |
| **구매 (MM)** | 발주, 입고 처리, 거래처 관리 |
| **영업 (SD)** | 견적, 수주, 출고, 매출 관리 |

### 모듈 간 데이터 연결 흐름

```
[구매]           [재고]                  [회계]
 발주서 생성  →  입고 처리 → 재고 증가  →  매입 전표 자동 생성

[영업]           [재고]                  [회계]
 수주 등록    →  출고 처리 → 재고 감소  →  매출 전표 자동 생성
```

→ 사람이 각 단계마다 따로 입력하는 게 아니라, **하나의 행위가 연쇄적으로 반영**된다.

## 6. ERP 도입 효과 & 한계

### 효과

- 부서 간 데이터 불일치 해소
- 실시간 경영 현황 파악 가능
- 업무 프로세스 표준화
- 결산/마감 시간 단축
- 내부 통제 강화 (누가 언제 무엇을 했는지 추적 가능)

### 한계 & 단점

| 한계 | 설명 |
|------|------|
| **도입 비용** | 대형 ERP(SAP, Oracle)는 도입에 수억~수백억 소요 |
| **구축 기간** | 중견기업 기준 6개월~2년 소요 |
| **커스터마이징 어려움** | 기성 ERP는 자사 업무에 맞게 수정하기 어려움 |
| **사용자 저항** | 기존 업무 방식을 바꿔야 해서 직원들의 거부감 발생 |
| **데이터 마이그레이션** | 기존 데이터를 ERP로 옮기는 작업이 까다로움 |
